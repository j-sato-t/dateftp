# 入力内容の保存と復元実装プラン

Fyneが提供する標準機能である `Preferences` APIを利用して、入力内容の永続化（保存）と次回起動時の復元（ロード）を実装します。

## 概要
Fyneの `Preferences` APIを使用すると、OSごとの標準的な設定保存ディレクトリ（Windowsの場合は `AppData\Roaming` 以下など）に自動的にキーバリュー形式でデータが保存されます。

## 実装手順

### 1. アプリケーションIDの指定
Preferencesを使用するためには、アプリケーションに一意のID（通常はリバースドメイン形式）を設定する必要があります。`app.New()` を `app.NewWithID()` に変更します。

```go
// 変更前
// myApp := app.New()

// 変更後
myApp := app.NewWithID("com.github.dateftp")
```

### 2. 次回起動時の入力値の復元（ロード）
各ウィジェット（入力欄やラベル等）を初期化する際に、Preferencesから前回保存されたデータを読み込んで初期値として設定します。データが存在しない場合のフォールバック値（空文字など）も指定できます。

```go
prefs := myApp.Preferences()

hostEntry := widget.NewEntry()
hostEntry.SetText(prefs.StringWithFallback("host", ""))

portEntry := widget.NewEntry()
portEntry.SetText(prefs.StringWithFallback("port", ""))

// ...他の項目も同様...
// ダウンロード先ディレクトリの復元
selectedDir := prefs.StringWithFallback("downloadDir", "")
if selectedDir != "" {
	downloadDirLabel.SetText(selectedDir)
}
```

*注意:* パスワードなどのクレデンシャル情報をそのまま保存するかどうかは、セキュリティ要件に応じて決める必要があります。今回はローカルツールという前提でパスワードも保存対象とするか、パスワードのみ毎回入力を促すか、どちらかを選択します。

### 3. 実行時の入力値の保存（セーブ）
「ダウンロード開始」ボタンが押下され、入力値のチェックを通過した直後に、現在の入力内容をPreferencesに保存します。

```go
startBtn := widget.NewButton("ダウンロード開始", func() {
	// ... (入力チェック) ...

	// 設定の保存
	prefs.SetString("host", hostEntry.Text)
	prefs.SetString("port", portEntry.Text)
	prefs.SetString("user", userEntry.Text)
	prefs.SetString("pass", passEntry.Text) // 扱う場合
	prefs.SetString("rootPath", rootPathEntry.Text)
	prefs.SetString("downloadDir", selectedDir)

	// ... (ログクリア処理、ダウンロード実行処理) ...
})
```

## Fyne AppData（ローカルのファイル）削除について
開発中等に保存された内容を破棄して初期状態に戻したい場合は、Fyneが生成した設定ファイルを物理的に削除することでリセット可能です。
Windowsの例: `%APPDATA%\com.github.dateftp\preferences.json`
